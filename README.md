Tackleit is a full-stack, AI-powered job discovery platform meticulously engineered to automate and personalize the complex process of finding the perfect career opportunity. It directly addresses the overwhelming and manual nature of job hunting by providing users with a highly curated list of roles that precisely match their skills and ambitions.

Most users give a new product **30‚Äì90 seconds** to prove its value. Tackleit is designed to hook you within that window by delivering immediate, tangible results, keeping you engaged for the crucial first 3‚Äì5 minutes of exploration.

## ‚ú® Core Features

Tackleit is packed with features designed for a modern, efficient, and personalized job hunt:

-   **ü§ñ Hyper-Personalized Recommendations**: At its core, the platform leverages a sophisticated AI model that analyzes both a user's detailed, manually-entered preferences and the parsed contents of their uploaded resume to deliver hyper-personalized job recommendations. This dual-analysis approach ensures maximum relevance.
-   **üìÑ Intelligent Resume Parsing**: Instantly parse your resume (CV) to automatically fill out your skills, roles, and experience, saving you time and effort.
-   **üí∏ Freemium Model**: A flexible two-tier system designed to suit your needs.
    -   **Free Plan**: Get monthly AI job recommendations and resume parsing.
    -   **Pro Plan**: Upgrade to receive weekly recommendations and parsing, plus unlock all premium features.
-   **‚≠ê Pro User Dashboard**: Pro subscribers get access to an enhanced dashboard with **Saved** and **Applied** sections. Users can **drag and drop** job cards to easily track their application status, creating a personalized and organized job search pipeline.
-   **üí≥ Secure Subscription Management**: Seamless and secure "Pro" user subscription management powered by **Razorpay**.
-   **üë§ Unified & Secure Authentication**: Easy and secure sign-up and login using either traditional email/password with **OTP verification** or the convenience of **Google Sign-In**.
-   **‚öôÔ∏è Comprehensive User Preferences**: A detailed preferences page allows you to specify desired roles, tech stack, experience, location, salary, and more to fine-tune your job matches.
-   **üìä Google Sheets Export (Pro Feature)**: Pro users can instantly export their curated job lists to a Google Sheet in their personal Google Drive, saving hours of manual data entry.
-   **üé§ Enhanced UX with Speech-to-Text**: The contact form features a speech-to-text option for the message field, allowing for easier and more accessible communication.
-   **üì¨ Automated Notifications**: Receive automated email confirmations for significant actions, such as when a Pro user updates their preferences, ensuring you're always in the loop.

## üõ°Ô∏è A Privacy-First Approach

We understand that your career data is sensitive. In an era where data privacy is paramount, some users might feel apprehensive about a platform handling their resumes, preferences, and personal information, especially with integrations like Google Drive. Here‚Äôs our commitment to you:

-   **User-Controlled Data & Opt-Out**: You are in complete control. The Google Sheets integration is entirely opt-in and requires your explicit consent. If you choose to disable the sync, the application immediately **revokes its access token** via the Google API and **deletes the stored credentials** from your user profile. The application only ever requests the minimum necessary permission: to create a single file (`Tackleit Job Recommendations`) in your Google Drive. It cannot read, modify, or delete any other files.
-   **Data Encryption**: Sensitive information, particularly your account password, is not stored in plaintext. We use industry-standard hashing algorithms to ensure your credentials are secure.
-   **Transparent Operations**: We believe in transparency. Our code is open-source, and we encourage you to review it. We are committed to being upfront about what data we collect and how we use it to power the platform's features.
-   **No Unnecessary Data Storage**: Your resume is parsed on the server to extract relevant information but is not stored long-term without your consent. You decide whether to save the parsed data to your profile.

Our goal is to provide powerful, time-saving features while respecting your privacy and giving you full control over your data.

### Deployment Architecture

-   **Frontend**: The Next.js application is deployed and hosted on **AWS Amplify**, providing a CI/CD pipeline that automatically builds and deploys the site on every push to the main branch.
-   **Backend**: The FastAPI backend is containerized using **Docker**. A manual-trigger **GitHub Actions** workflow builds the Docker image, pushes it to the **AWS Elastic Container Registry (ECR)**, and then deploys it as an **AWS Lambda** function. An **AWS API Gateway** is configured to trigger the Lambda function, providing a stable, serverless API endpoint.
-   **Domain**: The production domain (`tackleit.xyz`) is registered and managed via **Hostinger**.
-   **Automated Scraping**: A separate, scheduled **GitHub Actions** workflow runs weekly to execute the Python web scrapers, ensuring the job database remains fresh.

For a more visual understanding of the project's complete end-to-end architecture, please visit the **/workflow** page on the live site.

## ‚öôÔ∏è Getting Started: Local Setup

Follow these steps to set up and run the project on your local machine.

### Prerequisites

-   **Node.js**: v20.x or later
-   **Python**: v3.9 or later
-   **MongoDB Atlas**: A free cluster is sufficient.
-   **Git**

### 1. Clone the Repository

```bash
git clone https://github.com/saipraneethkotyada/Tackleit.git
cd Tackleit
```

### 2. Backend Setup

```bash
# Navigate to the backend directory
cd backend

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`

# Install dependencies
pip install -r requirements.txt

# Create and configure the environment file
cp .env.example .env
```

**Configure your `.env` file** with the required API keys and secrets.

### 3. Frontend Setup

```bash
# Navigate to the frontend directory
cd ../frontend

# Install dependencies
npm install

# Create and configure the environment file
cp .env.local.example .env.local
```

**Configure your `.env.local` file** with the required variables.

### 4. Running the Application

1.  **Start the Backend Server**:
    ```bash
    cd backend
    uvicorn main:app --reload
    # The backend will run on http://localhost:8000
    ```

2.  **Start the Frontend Server**:
    Open a new terminal window.
    ```bash
    cd frontend
    npm run dev
    # The frontend will be available at http://localhost:3000
    ```
